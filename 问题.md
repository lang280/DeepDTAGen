# 问题记录

原则：最小化修改，只修复不修就会影响运行的 bug，其余保持原装。

## 已修复（影响运行的 bug）

### B1 · `model.py` PositionalEncoding 越界 + 设备不一致

**文件**：`model.py`，`PositionalEncoding.__init__` / `forward`  
**症状**：原始代码在 `__init__` 中将 pe 建在 CPU 上并直接存为 buffer；`forward` 中用 `self.pe[:x.size(0), :]` 索引，若序列长度超过 `max_len` 会越界；在多卡或 CPU→GPU 场景下也可能设备不一致。  
**修复**：拆出 `_build_pe()` 方法；`forward` 中检查长度并按需重建，同时 `.to(x.device)` 确保设备一致。

### B2 · `utils.py` `get_aupr` 缺少实现 + 缺少导入

**文件**：`utils.py`  
**症状**：原始 `get_aupr(y_true, y_pred)` 使用 `precision_recall_curve`+`auc` 实现，但 `training.py` 的调用方式（`get_aupr(np.int32(G > t), P)`）存在语义问题；补充了 `average_precision_score` 导入，并将 `get_aupr` 改为 3 参数版本 `(predictions, true_labels, threshold)` 与调用方保持一致。

### B3 · `training.py` `get_aupr` 调用签名不匹配

**文件**：`training.py`，`test()` 内  
**症状**：原始调用 `get_aupr(np.int32(G > t), P)` 使用的是旧的 2 参数签名，与 B2 修复后的 utils.py 不匹配，会 TypeError crash。  
**修复**：改为 `get_aupr(P, G, t)`（3 参数）。

### B4 · `training.py` `mse_loss.item()` 类型错误

**文件**：`training.py`，`experiment()` 评估打印处  
**症状**：`mse_loss` 是 numpy float，无 `.item()` 方法，每 20 个 epoch 必然 AttributeError crash。  
**修复**：`mse_loss.item()` → `float(mse_loss)`。

### B5 · `test.py` 导入不存在的模块

**文件**：`test.py`  
**症状**：原始代码 `from datasets import TestbedDataset`，但项目中无 `datasets.py`，直接 ModuleNotFoundError。  
**修复**：改为 `from utils import *`。

### B6 · `test.py` `get_aupr` 调用签名不匹配

**文件**：`test.py`  
**症状**：原始 `get_aupr((predictions > threshold).int(), labels)` 使用旧的 2 参数签名，与 B2 修复后的 utils.py 不匹配。  
**修复**：改为 `get_aupr(predicted, ground_truth, threshold)`。

### B7 · `generate.py` `config['device']` KeyError

**文件**：`generate.py`  
**症状**：原始代码 `data.to(config['device'])`，但 `config` 字典中没有 `'device'` 键，运行即 KeyError crash。  
**修复**：`config['device']` → `device`。

### B9 · `generate.py` 使用 `torch.utils.data.DataLoader` 无法 collate PyG 数据

**文件**：`generate.py`  
**症状**：`torch.utils.data.DataLoader` 的默认 collate 函数无法处理 `torch_geometric.data.Data` 对象，迭代第一个 batch 即 `TypeError: default_collate: batch must contain tensors...`。原始代码中此错先于 B7（`config['device']`）触发，被遮蔽。  
**修复**：`from torch.utils.data import DataLoader` → `from torch_geometric.loader import DataLoader`。

### B8 · `generate.py` `mkdir(parents=False)` 崩溃

**文件**：`generate.py`  
**症状**：`output_dir.mkdir(parents=False, exist_ok=True)`，若 `generated_results/` 父目录不存在则 FileNotFoundError crash。  
**修复**：`parents=False` → `parents=True`。
